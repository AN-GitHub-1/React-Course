178. Using async / await

JavaScript promises. Using async / await is an alternative way to avoid using
a lot of then() and catch() blocks with Promises. 

In synchronous operations tasks are performed one at a time and only when one is 
completed, the following is unblocked. In other words, you need to wait for a task 
to finish to move to the next one. 

In asynchronous operations, on the other hand, you can move to another task before 
the previous one finishes. This way, with asynchronous programming youâ€™re able to 
deal with multiple requests simultaneously, thus completing more tasks in a much 
shorter period of time.   

So INSIDE a function, it doesn't procede until an "await" call ends but OUTSIDE it,
the program will read other functions and continue. So the async function is saying
"don't wait for me, go on, I'll let you know when I'm finished".
